# 更新日志

本项目的重要变更都记在这里。格式参考 [Keep a Changelog](https://keepachangelog.com/zh-CN/)，版本号遵循 [语义化版本](https://semver.org/lang/zh-CN/)。

> 确定性始终成立（同 seed + 同指令 → 逐位可复现）。**1.2.0 起的幸运事件会引入新的随机抽取**，因此跨大版本的「同 seed 指纹」会变（属预期）；纯文本/数值改动则保持 RNG 流不变。旧存档一律自动兼容（缺的新字段按默认补齐）。

## [2.2.3] — 2026-06-26

潜水「大遗迹」的触发是个设计 bug：它本该有自己独立的概率，却被错挂在「幸运事件」(5%) 下面，和小奇遇共抢一个名额，导致大场面遗迹太罕见（实测每 ~69 条成功渔获才撞 1 次）。

### 修复
- **大遗迹改为独立掷骰**：潜水时每条成功渔获先单独判一次「大遗迹」(`RUIN_CHANCE`，**3%**)，与下面的幸运事件互不影响；水下小奇遇（珊瑚宫/古遗迹等）仍留在幸运事件池里。大遗迹不再和小奇遇共用那一个 5% 名额。
- **全图鉴加成顺势重写**：`FULL_DEX_RUIN_BOOST=3` 现在直接乘在独立概率上，集齐图鉴后 **9%**（取代 2.2.2 里「在 luck 池内 ×3 权重」的旧写法）。

### 数值
- 未集齐图鉴：每条成功渔获撞大遗迹 **3.0%**（≈ 每 33 条 1 次；旧版 ~1.4%）。
- 全图鉴后：**9.0%**（≈ 每 11 条 1 次；旧版 ~2.7%）。
- 体感：一趟潜 10 条成功渔获，未集齐时约 26% 会撞上 1 次、全图鉴后约 61%。

### 说明
- **潜水随机流改变**（多了一次独立掷骰），同 seed 的潜水序列与 2.2.2 不同，属预期；**水面钓鱼随机流逐位不变**，旧存档一律兼容。
- 氧气瓶携带数不设上限（点数够即可囤），单条 `dive N` 深度仍封顶 20 步。

## [2.2.2] — 2026-06-25

试玩反馈两则：贵饵钓上垃圾、全图鉴后没事干。

### 变更
- **黄金亮片重做**：原先只抬 rare+ 权重、**完全不压普通鱼**，且在没有稀有鱼的水域加成全打空，玩家花大价钱却只少了点杂物。现给 `rarity_weight_mult` 加 `common: 0.5, uncommon: 0.8`（保留 rare 1.4 / epic 1.6 / legendary·mythic 2.0 与杂物 ×0.7）。满档水域稀有及以上占比 **11.4% → 18.1%**、普通鱼 66% → 52%；亮片真正偏向大货。文案也改准（「压住普通小鱼的咬口、把机会让给稀有及以上的稀客……这片水域有稀有鱼时才划算」），免得新手在没有稀有鱼的起步水域乱用。
- **全图鉴后大遗迹更常出**：集齐全部鱼种后，潜水时「大遗迹」(branch 奇遇) 权重 ×3（`FULL_DEX_RUIN_BOOST`）。没新鱼可发现的残局转向探遗迹——单次下潜撞大遗迹概率约 **1.4% → 2.7%**（实测约翻倍）。未集齐前完全不变。

### 说明
- 都是数值/文案改动，**RNG 流不变**（不新增随机抽取）；同 seed 的渔获结果会变（权重变了，属预期）。旧存档一律兼容。
- 大遗迹加成只在**图鉴 100%** 时生效，且只抬 branch 奇遇权重、不改 luck 触发率；没集齐的存档潜水行为逐位不变。
- 没有稀有鱼的水域（如芦苇河）亮片仍无意义——无稀有可抬，只剩减杂物；这是水域鱼群构成决定的，文案已提示。

## [2.2.1] — 2026-06-25

试玩反馈打磨（codex / GPT 盲玩 + 实测）：修结算、潜水去水面事件、播报精简、🎉 归位。

### 修复
- **远征结算少报渔获/新种**：之前结算只统计每竿主渔获，**分裂鱼钩/渔获热潮/分支鱼奖励额外钩上的鱼没算进去**（价值与图鉴没丢，只是结算漏报）。现在一律从本趟库存增量重算，全部计入。
- **潜水不再复用水面幸运事件**：分裂鱼钩/点石成金/渔获热潮/**河神祝福**/涨潮/蚌中生珠不再在潜水触发——在水下出戏，「河神祝福」还会凭空塞鱼饵 + 残留 `free_bait`。潜水现在只出**水下奇遇 / 大遗迹**（奇遇也因此露脸更勤）。

### 变更（播报精简：砍重复系统信息，保留氛围与手感）
- **新发现版式更紧**：去掉重复的「图鉴新发现」标签，压成 `🆕 名 · 稀有度 · 尺寸 · 价值` + **图鉴描述** +（水下鱼加一行 `🫧 捕获手感`）+ `首次收录 +N点`。描述与手感都保留。
- **🎉 留给图鉴里程碑**：普通新发现与传说/神话（有 👑/❖ 演出）都不加 🎉。🎉 用于**某档集齐**「🎉「常见」鱼已集齐！」、**每 10 种**「🎉 图鉴达成 N/81 种！」、**全收集**「🎉🎉 全图鉴收集完成！」。
- **停止原因只说一次**：去掉开头「停因」，远征结尾统一一行 `🤿 远征中止：发现新种｜潜 2/5｜余氧 3`；结算名单新种加 `★`。
- **去掉与 JSON 重复的人类状态行 / 实例编号 `[c_001]` / 咬钩过场**（保留入水氛围与手感）。
- **远征结算压成三行**：`🤿 …｜潜 X/Y｜余氧 Z` / `🐟 渔获 N：名单(★新)｜估值…｜🎁…` / `💰 收益 − 氧气 = 本趟 ±…`。
- **`encyclopedia` 不再刷 ？？？**（只列已发现）；**鱼饵反馈** `🎣 夜光饵偏好命中 N 条`。
- **`sell all` 提示宝物**（💎 用 sell item 单独卖）；**氧气货架暗示**「多带几瓶才够应对深处大遗迹的抉择」；大遗迹**保持稀有**（不调概率，撞见即惊喜）。

### 说明
- 播报/文案改动**保留原有随机抽取**（咬钩不显示仍抽一次），渔获逐位一致；唯「潜水去水面幸运事件」改了潜水随机池（同 seed 潜水指纹与 2.2.0 不同，预期）。**水面随机流逐位不变**，旧存档兼容。

## [2.2.0] — 2026-06-25

潜水重做（二）：大遗迹分支抉择 + 远征交互。

### 新增
- **远征交互**：`dive [N]` 开一趟远征（带 N 瓶氧气当深度预算）。途中遇「大遗迹」会**立刻暂停**，返回现场叙事 + 编号选项；用 **`choose <编号>`** 抉择，**`surface`** 主动上岸。远征暂停时只能 choose/surface（或只读指令），其余指令按下。
- **5 个大遗迹**（由现有奇遇升级）：深渊裂隙 / 沉船墓场 / 海底宝库 / 献祭石坛 / 龙王宫阙。每个 3 个选项、**消耗不同氧气**——氧气不够的选项变灰、只能放弃（取舍 & 挫败感）。其余 9 个奇遇仍是快速拿、不打断刷量。
- **分支收获**：选项可给点数 / 宝物 / **遗迹专属新鱼种**（`fish` 奖励，平时钓不到、只从遗迹得）/ 氧气 / 宝箱，或纯故事。新增 4 件宝物 + 4 条分支专属鱼（裂渊盲鳗 / 银鳞闪鱼 / 坛灵鲷 / 鳞甲守卫），总鱼种 77→81。
- `_grant_rewards` 新增 `fish` 通用奖励类型；分支专属鱼带 `branch_only`，不进普通水下池、不计「待发现」。

### 说明
- 本版重构了潜水的随机流（大遗迹改为暂停而非即时结算 + 新内容），同 seed 的潜水指纹与之前不同（预期）；**水面抛竿的随机流逐位不变**，旧存档自动兼容（缺字段默认补齐、进行中的远征可继续 choose/surface）。

## [1.8.0] — 2026-06-25

试玩反馈调整：潜水后期化 + 早期经济 + 氛围不刷屏。

### 变更
- **氧气瓶后期上架**：解锁第一个潜水点之前，商店不卖氧气瓶、也买不了（潜水是后期玩法，前期专心钓鱼集藏宝图）。解锁后自动上架，老板台词也跟着变。

- **氛围句只换场景出一次**：同一地点 + 季节里反复抛竿不再重复刷环境氛围句；换地点或季节流转后才再出一句（存档新增 `ambience_scene` 标记当前场景）。

### 说明
- 氛围改动减少了抛竿时的随机抽取，同 seed 指纹与之前不同（预期）；旧存档兼容、缺字段默认补齐。

## [1.7.0] — 2026-06-25

藏宝图碎片的更多来源 + 稀有「完整藏宝图」。

### 新增
- **完整藏宝图（稀有奖励）**：直接解锁一个尚未解锁的潜水点，跳过集碎片。
- **碎片 / 藏宝图新来源**：除了水面钓鱼，现在漂流瓶和各类宝箱也能开出——
  - 漂流瓶：必给 1 块碎片。
  - 锈迹宝箱 / 藤壶密箱 / 船长遗箱（前期就能钓到的水面宝箱）+ 海底宝库：掉落表新增碎片（常见）与完整藏宝图（稀有）。
- 碎片优先补**当前地点**（若其潜水未解锁），否则随机一处未解锁地点；完整藏宝图随机解锁一处。潜水点全解锁后，碎片 / 藏宝图自动折算成点数、不浪费。

### 实现
- `_grant_rewards` 新增 `fragment` 与 `map` 两种通用奖励，所有发奖路径（事件 / 宝箱 / 奇遇）共用。
- 本版给漂流瓶 / 宝箱奖励加了随机抽取，**同 seed 指纹与之前不同**（新玩法预期）；自身完全确定，旧存档兼容。

## [1.6.0] — 2026-06-25

潜水点解锁进程：藏宝图碎片。给「无脑潜」加一层前置养成。

### 新增

- **藏宝图碎片**：在某钓点**水面钓鱼**时，15% 概率额外捞到一块「该地的藏宝图碎片」；集齐 N 块（按水域深浅 3 / 4 / 5）**自动拼成藏宝图**，解锁这里的潜水点。碎片按地点独立累积——想在哪潜，就先在哪钓。

### 兼容
- **老存档自动派生**：已在图鉴里收录过某地水下鱼的钓点，视为已解锁（不锁老玩家）。
- 本版给水面抛竿增加了一次「碎片」随机抽取，故同 seed 的**水面指纹与之前版本不同**（属新玩法预期）；新版自身完全确定，旧存档照常加载、缺字段按默认补齐。

## [1.3.0] — 2026-06-25

潜水系统：买氧气瓶下水，捕获只有水下才有的鱼种。

### 新增
- **氧气瓶**（shop 新增 `oxygen`，45 点/瓶）：潜水消耗品，独立库存，不占鱼饵。`buy oxygen [数量]`。
- **`dive` 动作**：`dive [次数] [stop=new,rare,event]`。每潜一次耗 1 氧气瓶、**不耗鱼饵**；带几瓶就能连潜几次（同连钓，回一个汇总）。体型/价值/图鉴/幸运事件照常，水下不出漂流瓶/宝箱。
- **22 种水下鱼种**（`dive: true` + tags 含 `underwater`，每钓点 2 种，跨常见~传说）：只在潜水时出现，水面抛竿**永远钓不到**；反之潜水也不出水面鱼。并入总图鉴（55→77）。每条带 **`capture_feel` 捕获手感**——钓上来与 `look` 时多显示一行「🫧 手感：…」的触觉/通感描写；`look` 还标「🤿 潜水鱼，水面钓不到」。
- **`goto` 水下提示**：买过氧气瓶后，钓点清单与进入钓点会多显示「🤿水下 N 种待发现」（没买过不显示，免剧透）。
- 存档新增 `oxygen` / `oxygen_ever` 与 `stats.total_dives`（旧档自动补默认）；状态栏 JSON 在有氧气时多 `oxygen` 字段；`status` 显示氧气瓶。

### 说明
- 水下鱼按 `dive` 标记从水面池子里剔除，因此**水面抛竿的随机流与 1.2.0 逐位一致**——老存档、水面回放完全不受影响；潜水是另一条独立的确定性路径。

## [1.2.0] — 2026-06-25

幸运随机事件：钓到鱼时小概率触发的高光时刻，给抛竿加一层惊喜。

### 新增
- **6 种幸运随机事件**（成功钓到鱼后 5% 概率触发，立即生效或给后续几竿挂 buff）：
  - **分裂鱼钩**：鱼钩一分为三，这一竿再多上两条。
  - **点石成金**：这条鱼价值 ×3。
  - **渔获热潮（fever）**：接下来钓到的 3 条鱼全部数量翻倍。
  - **河神的祝福**：退还这一竿的饵，接下来 3 竿不耗鱼饵。
  - **千载难逢的涨潮**：这条直接涨到该鱼种的极限体型（破纪录大鱼）。
  - **蚌中生珠**：鱼肚里掏出一枚随机财宝。
- 存档新增 `fever` / `free_bait` 两个 buff 计数（旧存档自动补 0）；状态栏 JSON 在生效时多 `fever` / `free_bait` 字段。连钓汇总里，触发幸运事件 / 热潮翻倍的那竿会留完整文案、不被折叠。

### 说明
- 幸运事件**只在成功钓到鱼时**掷骰（杂物 / 空军 / 漂流瓶等不触发），与既有漂流瓶/宝箱事件相互独立。
- 引入了新的随机抽取，故 1.2.0 与之前版本的「同 seed 指纹」不一致（属预期）；1.2.0 自身仍完全确定（同 seed + 同序列逐位可复现）。

## [1.1.0] — 2026-06-25

省 token + 多指令一次跑 + 状态栏 JSON + 钓点「待发现」计数。一轮里给 AI 的信息更密、来回更少，玩起来更顺。

### 新增
- **叠加指令（batch）**：用 `;` 或换行把多条不同指令串成一批、一次按顺序执行（最多 8 条），如 `buy basic_worm 10; cast 10`、`goto reed_river; cast 8 stop=new`。每段前缀 `▶ 指令` 小标题；某条出错只影响那一条、不打断后面的。把「买饵→抛竿」「换地点→抛竿」这类常见组合从多次往返压成一次。（`tool-schema.json` 同步加了 `batch` 动作与 `steps` 数组。）
- **状态栏 JSON**：每次 `cmd()` 返回的末尾都附一行紧凑机读状态——`📊 {"pts","loc","sea","turn","enc","bait","hold"[,"chest"]}`（点数 / 地点 / 季节 / 回合 / 图鉴进度 / 余饵 / 未卖渔获 / 待开宝箱）。AI 看这一行就够、不必再单独 `status`，省一次往返。
- **钓点「待发现」计数**：`goto` 列表与进入钓点时，显示该点**当季还有几种没见过的鱼**（如「本季待发现 6 种」），指引补图鉴。**传说/神话单列**为「（+N 传说级）」，且只算该地点专属的（排除全域出没的神话，免得每个钓点都被顶高、也免常规墙永远凑不满）。

### 变更
- **连钓提示更醒目**：单竿抛竿成功后追加一行 `💡` 提示，引导用 `cast 10` 连钓 + `stop=new/rare` 早停，减少一竿一条的来回。
- **抛竿省 token**：杂物 / 空军这类**无演出的填充竿去掉重复的状态行**（状态统一交给末尾的 `📊` JSON）；钓到鱼（尤其新发现）的完整演出——咬钩过场、鱼卡、`★图鉴新发现`、首次收录奖励、图鉴进度行——**保持不变**。连钓汇总也去掉了末尾重复状态行。
